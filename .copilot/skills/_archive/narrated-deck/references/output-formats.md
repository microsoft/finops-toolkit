# Output formats

The skill produces four artifacts per source doc. This document defines the schema for each.

## scenes.json

Machine-readable scene plan. Single JSON object:

```json
{
    "h1": "FinOps toolkit SRE Agent",
    "doc_path": "/path/to/source.md",
    "scenes": [
        {
            "id": 1,
            "heading": "Intro",
            "layout": "TITLE",
            "text": "verbatim prose from the source, markdown-stripped",
            "raw_markdown": "the original markdown of this section",
            "has_table": false,
            "has_code": false,
            "narration_only": false
        },
        {
            "id": 2,
            "heading": "What you get",
            "layout": "TABLE",
            "text": "...",
            "raw_markdown": "## What you get\n\n| Component | ... |",
            "has_table": true,
            "has_code": false,
            "narration_only": true
        }
    ]
}
```

Field semantics:

| Field | Type | Meaning |
|---|---|---|
| `id` | int | 1-based scene index. Drives filename ordering. |
| `heading` | string | The H2 heading text (or "Intro" for scene 1). |
| `layout` | enum | One of `TITLE` `BULLETS` `TABLE` `CODE` `CALLOUT` `OUTRO`. |
| `text` | string | Cleaned, narration-bearing prose. Markdown stripped, links flattened, code backticks removed, bullets folded into "label: text" form. Verbatim author voice. |
| `raw_markdown` | string | The original markdown for the section. Used by `build_pptx.py` to render tables and code blocks faithfully. |
| `has_table` | bool | True if section contains a markdown table. |
| `has_code` | bool | True if section contains a fenced code block. |
| `narration_only` | bool | True if the section has no spoken-worthy prose (table or code dominates with under ~20 words of explanation). The transcript file for these scenes is a single-line marker, not real narration. |

## transcripts/scene_NN_<slug>.txt

One file per scene. Plain UTF-8 text. Filename pattern: `scene_<id-zero-padded-2>_<heading-slug>.txt`.

For prose-bearing scenes, the file contains the normalized narration with paragraph breaks marked as SSML `<break time="1300ms" />` tags:

```
The FinOps toolkit SRE Agent is AI-powered FinOps and capacity management automation built on Azure SRE Agent. <break time="1300ms" /> The agent focuses on three core design principles. Automate the rhythm. Ground every answer. Act with experts.
```

For slide-only scenes (tables, code, anything with `narration_only: true`), the file is a marker:

```
# slide-only — no narration
# heading: What you get
# layout: TABLE
```

Downstream TTS skills should skip files starting with `# slide-only`.

The `<break time="1300ms" />` tag is recognized natively by:
- ElevenLabs (treats as 1.3s pause)
- Azure Speech Service (treats as 1.3s pause)
- Most other SSML-compatible engines

For engines that don't speak SSML, the tag renders as visible text — agents should pre-process by stripping `<break ... />` tags or pass `--no-breaks` to `transcript.py` at generation time.

## deck-outline.md

Human-editable scene table in the same 7-cell format that `src/templates/sre-agent/training/release-deck/build.py` consumes. This means the same outline can be rebuilt by either:
- This skill's `build_pptx.py`
- The hand-tuned release-deck builder

```markdown
# FinOps toolkit SRE Agent — narrated deck outline

Generated by `narrated-deck` skill from `docs-mslearn/toolkit/sre-agent/overview.md`.
Edit this table by hand to override scene titles, content, layouts, or assets,
then rebuild slides with `build_pptx.py --outline <this-file>`.

| num | stage | title | content | notes | layout | screens |
|---|---|---|---|---|---|---|
| 1 | overview | FinOps toolkit SRE Agent | • Subtitle bullets | The FinOps toolkit SRE Agent is...<br/><br/>... | kind=TITLE | None |
| 2 | overview | What you get | • Component table | (slide-only — no narration) | kind=TABLE | None |
```

Cell semantics:

| Cell | Meaning |
|---|---|
| `num` | Scene id (matches `scenes.json` and the transcript filename). |
| `stage` | Doc slug. Used as a chyron/cluster label in PowerPoint. |
| `title` | Slide title. For the `TITLE` layout, this is the doc H1. |
| `content` | Slide body bullets. `<br/>`-separated. Each bullet starts with `• `. For tables/code, this can be summary text since the slide renders the actual table/code. |
| `notes` | PowerPoint speaker notes = the narration. `<br/><br/>` between paragraphs. This is the source of truth for narration when the deck is hand-edited. |
| `layout` | `kind=TITLE` / `kind=BULLETS` / `kind=TABLE` / `kind=CODE` / `kind=CALLOUT` / `kind=OUTRO`. |
| `screens` | Asset references (e.g. `assets: image=foo.svg`) or `None`. |

To override: edit the table. Then run `build_pptx.py --outline deck-outline.md --out slides.pptx` (planned — not yet implemented; current `build_pptx.py` reads `scenes.json` only). For now, override by editing `scenes.json` directly.

## slides.pptx

Editable PowerPoint deck. Inherits the bundled `assets/source-template.pptx` master slides for brand consistency.

Slide structure:
- One title slide (`TITLE` layout) for scene 1
- One slide per H2 in the source doc, layout chosen by classifier
- One outro slide at the end (`OUTRO` layout)

Each slide carries the narration in the speaker-notes pane. Open in PowerPoint, switch to Notes view, and you'll see slide + spoken script side by side.

The deck is fully editable. Renders to PDF, PNG, or video via the standard PowerPoint or LibreOffice export paths.
