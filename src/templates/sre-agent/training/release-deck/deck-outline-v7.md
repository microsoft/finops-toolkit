> **Spine.** Full Azure estate lifecycle, capacity leads, cost partners. Six lifecycle stages plus a meta-stage that closes the loop back to the agent itself. The audience walks the agent across the customer's estate from forecast through decommission, and at every stage sees capacity and cost reasoned about together.

> **Audience.** Internal Microsoft FTK community + adjacent MCAPS field roles (sellers, SE, CSA, partner architects). Peer-to-peer.

## Lifecycle and equal-billing rules

**The six stages form a cycle.** Stage 6 (Optimize / decommission) feeds back into Stage 1 (Forecast) when utilization drift triggers a re-forecast. Show as a wheel diagram in the recap slide; Stage 7 (Improve the agent) sits in the middle as the hub.

**Capacity leads at every stage.** The opening sentence on every operational slide states the capacity question. The cost question follows. The agent's combined answer is the third element. This isn't artificial; capacity is the gating constraint at every stage — unlimited budget can't deploy without quota, but quota without commitment is full-price capacity.

**Capacity verb varies per stage** so the rhythm doesn't read as rote:

| Stage | Capacity verb | Cost verb |
|---|---|---|
| Forecast | will we have it? | what will it cost? |
| Procure | should we buy it? | should we commit to it? |
| Allocate | is it where we need it? | does it roll up cleanly? |
| Deploy | can we use it right now? | what's the budget impact? |
| Operate | how is it being used? | what's the spend trend? |
| Optimize | can we release it? | can we recover the spend? |

**Per-stage slide template** (varies in slide count by stage importance):

1. **Stage opener** — stage name on the wheel diagram + two-column anchor (capacity question / cost question) + tagline.
2. **Worked example** — capacity question first, cost question second, agent's combined answer. Same screenshot template both columns.
3. **Scheduled tasks at this stage** — capacity tasks first, cost tasks second, with cadence labels.
4. **Why it matters** — three short outcome cards: capacity outcome, cost outcome, lifecycle-loop outcome (what feeds the next stage).

The Deploy stage breaks template once with the 8-check pre-deploy gate slide. Stage 7 breaks template differently with 'gaps you found / PRs your team can ship' replacing 'capacity question / cost question.'

## Voice and presentation rules

1. **Sentence case** for every title, header, table column, and bullet.
2. **Second person.** "You" addresses the field reader. "Your customer" addresses the deployment owner.
3. **No first person.** No "I". No "we" except when speaking as Microsoft to the audience, sparingly.
4. **No proper-noun customer attributions.** Never name a customer organization in slides or speaker notes.
5. **No quotes from any specific person.**
6. **Customer-outcome verbs only.** Runs, returns, reports, recommends, answers, delivers, supports. Avoid transforms, eliminates, empowers, unleashes, retires (as in "retires standing meetings" — reword to "replaces the data-gathering work that drives standing meetings").
7. **Active voice, present tense.**
8. **No write-op promises.** v1 reads, recommends, reports.
9. **Capacity leads sentence-by-sentence** within every operational slide. Cost has a substantive paired question. SRE Agent is the runtime, not the headline.
10. **Vary the capacity verb per stage** (will we have it / should we buy it / is it where we need it / can we use it now / how is it being used / can we release it).
11. **Speaker notes target 60–90 seconds each.**
12. **Same template both sides.** Capacity / cost slide pairs use identical layouts; capacity card sits in the leading position.
13. **Don't prescribe.** The deck shows what the agent does and how customers extend it. It does not teach the FinOps Framework, the capacity supply chain, or the FinOps practitioner role. The audience already runs their own.
14. **Gaps as PRs.** Honestly acknowledge the two scheduled-task gaps (Allocate cost-side, Optimize both sides) within their stages, AND repurpose them in Stage 7 as concrete first-PR examples. Gaps prove the open-source loop closes; they're not deficiencies.
