# [[FinOps toolkit SRE Agent]] release training deck outline (V8)

> **Audience.** [[MCAPS]] worldwide — AE / SE / CSA / CSAM / partner architects. Includes the existing [[FinOps Toolkit]] community in MCAPS. **L300/L400 density.** Not a cold-pitch room.

> **Canonical ask source.** `memory://work/mcaps/mcaps-internal-asks-finops-toolkit-azure-capacity-ai-cost-may-2026`. **Every body slide cites at least one ask number** in its layout directive (`addresses=N,N`) AND in its speaker notes (full memory URI + verbatim ask reference). The 73-ask table in that memory note is the single source of truth — do not re-derive.

> **Structure.** Two parts × ~45 min, each ~30-40 slides. Each ask cluster = 3 slides (A/B/C = **ASK / SHOW / TELL**). **A = ASK** (verbatim customer quote, low density, big type). **B = SHOW** (the answer — named tools/tasks + chart from a real run, eye candy carries the slide). **C = TELL** (the L400 details — YAML, cron, KQL, runtime, Monday move; code carries the slide). Plus Part 3 wrap (~8 slides) for honest-not-solved cluster + close.

> **Build mechanics.** New layout kinds: `ASK_A`, `ASK_B`, `ASK_C` + per-row directive `addresses=N,N verdict=green|yellow|red` parsed by build.py. Renderer emits ask-coverage footer chip on every body slide. Speaker-notes citation format below.

> **Speaker notes citation format (mandatory on every body slide).**
> ```
> Sources: MCAPS ask #N — "<verbatim ask theme from canonical table row>"
> Canonical: memory://work/mcaps/mcaps-internal-asks-finops-toolkit-azure-capacity-ai-cost-may-2026
> Coverage verdict per canonical: 🟢 ships-today | 🟡 partial | 🔴 out-of-scope
> Evidence pack: src/templates/sre-agent/training/release-deck/evidence/tasks/<task>.md
> Chart proof: src/templates/sre-agent/training/release-deck/charts/svg/<chart>.svg
> ```
> The build script extracts the citation block to render the footer chip; the rest stays in speaker notes for the field to follow back to source if needed.

## Voice and presentation rules (carry from V6 — non-negotiable)

1. **Sentence case** for every title, header, table column, bullet.
2. **Second person.** "You" addresses the field reader. "Your customer" addresses the deployment owner.
3. **No first person.** No "I". No "we" except sparingly when speaking as Microsoft.
4. **Verbatim asks may name accounts** (Snowflake INT, Aon, Synopsys, Trimble) since they're public internal corpus citations from the canonical memory note. **Body slide content does NOT name customers.**
5. **No outcome verbs.** Banned: surfaces, transforms, empowers, unleashes, retires, eliminates. Use named tools, named tasks, named subagents, named runtimes, named API surfaces.
6. **Active voice, present tense.**
7. **Honest scope on write ops.** v1 has 6 write-capable tools (`deploy-budget`, `deploy-bulk-budgets`, `deploy-anomaly-alert`, `deploy-bulk-anomaly-alerts`, `suppress-advisor-recommendations`, `resource-graph-query`) — name them where relevant; don't hide them, don't over-claim.
8. **Speaker notes target 60-90 seconds each.**
9. **Slide A = ASK** (verbatim customer quote, low density, big type — speaker reads aloud and pauses). **Slide B = SHOW** (the answer with chart/Teams-card eye candy + named tools — proof carries the slide). **Slide C = TELL** (L400 details — YAML, cron, KQL, runtime, Monday-move command; code carries the slide).
10. **Honest scope** is a credibility weapon, not a weakness. The 25 🔴 asks in the canonical table get one cluster of slides naming the actual owner per ask group.
11. **Every body slide has an `addresses=N,N verdict=green|yellow|red` directive in its layout cell** AND **a citation block at the end of speaker notes** referencing the canonical memory URI + ask number(s) + evidence pack path.
