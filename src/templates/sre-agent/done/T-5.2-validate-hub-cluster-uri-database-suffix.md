# T-5.2 Validate hub cluster URI includes database suffix

**Pipeline**: lightweight
**Pipeline Digest**: a]lightweight
**Priority**: P0
**Parent**: session-1777654291702-12465

## Problem

The SRE Agent Bicep connector requires the FinOps Hub cluster URI to include a database name:
`https://cluster.region.kusto.windows.net/databasename`

`deploy.sh` accepts `--finops-hub-cluster-uri` but does not validate the suffix. When omitted,
the ARM deployment fails with:

```
InvalidRequestParameter: Kusto DataSource must include a database name
```

This wastes a full provisioning cycle (~2 min) before failing.

## Fix

1. In `deploy.sh`: after parsing args, validate `FINOPS_HUB_CLUSTER_URI` contains a path
   component (i.e., something after the hostname). If missing, append `/hub` as the default
   database name and log a warning.
2. In `deploy.ps1`: same validation logic in PowerShell.
3. Update `--help` text to document the required format.
4. Update README "Advanced/manual" section to show the `/hub` suffix in the example URI.

## Acceptance

- `deploy.sh --finops-hub-cluster-uri https://foo.kusto.windows.net` auto-appends `/hub` and logs warning.
- `deploy.sh --finops-hub-cluster-uri https://foo.kusto.windows.net/mydb` passes through unchanged.
- README examples all include the database suffix.
