# T-5.1 Fix CRLF line endings in shell scripts

**Pipeline**: lightweight
**Pipeline Digest**: a]lightweight
**Priority**: P0
**Parent**: session-1777654291702-12465

## Problem

`scripts/deploy.sh` and `scripts/post-provision.sh` ship with Windows CRLF (`\r\n`) line endings.
macOS default `/bin/bash` 3.2 cannot parse them:

```
set: pipefail: invalid option name
syntax error near unexpected token `{'
```

This blocks all macOS deployments. The scripts are the only deployment entrypoint.

## Fix

1. Convert `scripts/deploy.sh`, `scripts/post-provision.sh`, and `scripts/deploy.ps1` to LF.
2. Add `.gitattributes` in `src/templates/sre-agent/` enforcing `*.sh text eol=lf`.
3. Verify with `file scripts/*.sh` — must report "ASCII text" or "UTF-8 text" with no CRLF.

## Acceptance

- `file scripts/deploy.sh` shows no CRLF.
- `file scripts/post-provision.sh` shows no CRLF.
- `.gitattributes` enforces LF for `*.sh`.
- `bash scripts/deploy.sh --help` works on macOS `/bin/bash` 3.2.
