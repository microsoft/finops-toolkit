## Problem
The shipped azd parameter file is stale relative to `infra/bicep/main.bicep`. It still passes `resourceGroupName`, which `main.bicep` no longer declares, and it omits the new hub deployment parameters.

## Expected behavior
`infra/bicep/main.parameters.json` should only pass parameters declared by `main.bicep`, and deployment wrapper environment values such as `DEPLOY_FINOPS_HUB` and `FINOPS_HUB_DATA_EXPLORER_SKU` should reach Bicep when the shipped deploy path enables them.

## Evidence
Code review finding: `infra/bicep/main.parameters.json:11` passes `resourceGroupName`. `main.bicep` declares `environmentName`, `location`, `deployerPrincipalType`, `finopsHubClusterUri`, `DEPLOY_FINOPS_HUB`, `FINOPS_HUB_DATA_EXPLORER_SKU`, `adxClusterName`, and `adxClusterResourceGroupName`.
