# T-4.5 — Update post-provision.sh for new scheduled tasks

**Component**: `scripts/post-provision.sh`, `scripts/post-provision.ps1`

## Scope

Ensure post-provision scripts handle the 9 new scheduled tasks. The existing `apply_scheduled_tasks()` function already iterates `sre-config/scheduled-tasks/*.yaml`, so if the new tasks follow the same directory convention, no code changes may be needed — just verification.

Also verify the PowerShell equivalent (`post-provision.ps1`) handles the same.

## Acceptance criteria

1. `post-provision.sh` discovers and applies all 18 scheduled tasks
2. `post-provision.ps1` discovers and applies all 18 scheduled tasks
3. No hardcoded task names — both scripts use directory iteration
4. Tested locally with a dry-run or against the live agent
