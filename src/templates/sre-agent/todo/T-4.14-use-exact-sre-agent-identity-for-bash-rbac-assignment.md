## Problem
The Bash post-provision RBAC assignment can grant subscription roles to the wrong identity by selecting the first user-assigned identity in the resource group.

## Expected behavior
RBAC assignment should use the exact SRE Agent managed identity principal emitted by deployment outputs, and should fail clearly if that identity cannot be resolved. It must not infer broad subscription permissions from the first identity in the resource group.

## Evidence
Code review finding: `scripts/post-provision.sh:260` uses `az identity list --resource-group "$rg_name" --query '[0].principalId'`. `main.bicep` outputs the exact SRE Agent UAMI principal as `IDENTITY_PRINCIPAL_ID`, but the Bash script ignores that output.
