# T-5.5 Prohibit manual Azure resource intervention in AGENTS.md

**Pipeline**: lightweight
**Pipeline Digest**: a]lightweight
**Priority**: P0
**Parent**: session-1777654291702-12465

## Problem

Agents have repeatedly corrupted production Azure deployments by running srectl, az CLI, and other commands directly against live resources outside the deploy.sh/azd release process. This has happened multiple times in this session alone:

1. Agent dispatched srectl push against live SRE Agent without user approval
2. Partial/corrupted deployment required RG deletion and redeployment
3. No guardrail in AGENTS.md prohibiting this behavior

## Fix

Update `AGENTS.md` to explicitly prohibit any manual Azure resource intervention. All Azure changes must go through `deploy.sh` or `azd up` — the established release process. No ad-hoc az CLI, srectl, or ARM calls against live resources.

## Acceptance

- AGENTS.md contains an explicit prohibition on manual Azure resource intervention
- The prohibition is prominent (not buried at the bottom)
- References the release process (deploy.sh / azd up) as the only permitted path
