# T-4.2 — Create 9 new scheduled tasks from azcapman lab mapping

**Component**: `sre-config/scheduled-tasks/` (9 new task YAMLs)

## Scope

Create 9 new scheduled tasks derived from the azcapman labs (Forecast→Procure→Allocate→Monitor). Each task uses the deployed Python and Kusto tools.

## New tasks

| Task name | Source lab | Primary Python tools | Cadence |
|---|---|---|---|
| ComputeUtilizationTrend | lab-05-monitoring | vm-quota-usage, resource-graph-query | Weekly |
| StoragePaasGrowthForecast | lab-02-forecasting | non-compute-quotas, resource-graph-query | Monthly |
| SkuAvailabilityAudit | lab-04-procurement | sku-availability | Weekly |
| NonComputeQuotaAudit | lab-05-monitoring | non-compute-quotas | Weekly |
| BudgetCoverageAudit | lab-05-monitoring | deploy-budget, resource-graph-query | Monthly |
| AlertCoverageAudit | lab-05-monitoring | deploy-anomaly-alert, resource-graph-query | Monthly |
| MonitoringScopeValidation | lab-05-monitoring | resource-graph-query, data-freshness-check | Weekly |
| BenefitRecommendationReview | lab-04-procurement | benefit-recommendations | Weekly |
| AdvisorSuppressionReview | lab-05-monitoring | suppress-advisor-recommendations | Monthly |

## Acceptance criteria

1. Each task YAML follows the existing scheduled task schema (api_version, kind, metadata, spec)
2. Each task has Phase 0 (read knowledge base), Teams delivery, and knowledge/Teams split
3. Each task prompt names the specific Python tools to use with `client_id` parameter
4. All task cron schedules are valid and non-overlapping with existing 9 tasks
5. Tasks deploy via `srectl scheduledtask apply` without errors
6. Repo tests updated to validate new task YAMLs exist
