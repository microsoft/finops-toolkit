## Problem
The Windows azd hook invokes `scripts/post-provision.ps1`, but subscription-level RBAC assignment was moved out of Bicep into post-provision scripting and appears to exist only in the Bash implementation.

## Expected behavior
The Windows/PowerShell deployment path should assign the same required subscription-scope Reader and Monitoring Contributor permissions promised by the template, or invoke a shared implementation that does so.

## Evidence
Code review finding: `azure.yaml:18-20` invokes `pwsh -NoProfile -File ./scripts/post-provision.ps1` on Windows. The reviewed PowerShell post-provision flow lacks Reader/Monitoring Contributor role assignment or `IDENTITY_PRINCIPAL_ID` handling, while `main.bicep` states subscription RBAC is handled by post-provision scripts.
